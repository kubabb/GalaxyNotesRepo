MASTER PROMPT: OPERACJA „GALAXY-PILOT EVOLUTION v2.0"

Status: Strategia HERMES. Rozbudowa systemu o uniwersalny importer, pełne sterowanie lotem 3D, interaktywny HUD oraz automatyczną kontrolę wersji.
Model: Gemma 4 (google/gemma-4-26b-a4b-it:free) z retry logic i fallbackiem lokalnym.

================================================================================
ZESPÓŁ AGENTÓW (8 członków + HERMES jako strateg):
================================================================================

1. SECURITY-OFFICER   – Strażnik bezpieczeństwa. Pilnuje .env, kluczy API i .gitignore.
2. ASTRONOMER         – Matematyk galaktyki. Fizyka semantyczna – pozycje na podstawie powiązań.
3. STORYTELLER        – Deep Scan AI. Analiza kontekstowa, auto-[[WikiLinks]], retry logic.
4. CODER              – Inżynier kokpitu 3D. WASD+mysz, Hyperjump, System Zoom, HUD z treścią.
5. WATCHMAN           – Strażnik procesu. Monitoring, logi, wznowienie, timeouty.
6. GIT-PUSHER         – Agresywny deployment. Commit+push po każdym pliku. Kopia zapasowa na bieżąco.
7. DEBUGGER           – QA i diagnosta. Retry logic API, stabilność, naprawa timeoutów.
8. LIBRARIAN          – Bibliotekarz. Struktura .galaxy_map/, BRAIN, drag&drop importer.
9. HERMES             – Strateg. Koordynuje agentów, podejmuje decyzje architektoniczne.

================================================================================
FAZY PROJEKTU (EVOLUTION v2.0)
================================================================================

FAZA 0: IMPERIUM HERMESA (Strategia)
   - HERMES aktualizuje plan.txt i rozdziela zadania.
   - Definiuje interfejsy między agentami.

FAZA 1: IMPORTER I BEZPIECZEŃSTWO (SECURITY-OFFICER + LIBRARIAN)

   a) Dynamiczny Importer:
      - Użytkownik wskazuje dowolny folder (TARGET_PATH) przez:
        * Ścieżkę w setup.ps1
        * Drag&Drop pliku/folderu na okno przeglądarki
        * Wybór z listy w GUI kokpitu 3D
      - LIBRARIAN skanuje TARGET_PATH i buduje .galaxy_map/ lustrzane.

   b) Struktura Lustrzana v2:
      - Każdy plik z TARGET_PATH → .galaxy_map/[ścieżka].md
      - Pliki .md zawierają: YAML frontmatter, metadane Sci-Fi, linki, treść.
      - Jeśli projekt "na surowo" – LIBRARIAN nadaje strukturę BRAIN.

   c) Bezpieczeństwo:
      - SECURITY-OFFICER waliduje .env przy każdym starcie.
      - .gitignore blokuje: config/.env, __pycache__/, data/output/, logs/.
      - DEBUGGER skanuje wycieki kluczy przed każdym pushem.

FAZA 2: DEEP SCAN AI (STORYTELLER + DEBUGGER)

   a) Analiza Gemma 4:
      - Deep Scan każdego pliku z TARGET_PATH.
      - Retry logic: 3 próby z backoff (2s, 4s, 8s) przy timeout/429.
      - Fallback lokalny: jeśli API padnie – heurystyka tekstowa.

   b) Auto-WikiLinks:
      - STORYTELLER proponuje [[WikiLink]] na podstawie kontekstu.
      - LIBRARIAN zapisuje propozycje w .galaxy_map/.
      - ASTRONOMER używa linków do fizyki galaktyki.

   c) Diagnoza DEBUGGERA:
      - Analiza dlaczego API wywalało timeouty.
      - Implementacja queue + batch processing.
      - Limit równoległych requestów: max 3 na raz.

FAZA 3: FIZYKA SEMANTYCZNA (ASTRONOMER)

   a) Grawitacja Węzłów v2:
      - Pozycje (X,Y,Z) na podstawie:
        * Ilości linków przychodzących (masa)
        * Odległości semantycznej (cosine similarity tekstów)
        * Daty modyfikacji (gwiazdy aktywne bliżej centrum)
      - Galaktyka spiralna z 3 ramionami.
      - Konstelacje = gęsto połączone klastry.

   b) Live-Sync:
      - Watcher dodaje nowe gwiazdy w locie.
      - ASTRONOMER przelicza pozycje przy nowym pliku.

FAZA 4: KOKPIT 3D v2 (CODER + DEBUGGER)

   a) Model Lotu (Free-Roam):
      - WASD: Swobodne latanie w przestrzeni 3D.
      - Mysz: Obrót kamery (pitch/yaw).
      - Shift: Hyperjump – płynny dolot do zaznaczonej gwiazdy.
      - Space: System Zoom – powrót do widoku całej galaktyki.
      - Scroll: Przyspieszenie/zwolnienie.

   b) Interaktywny HUD v2:
      - Po zbliżeniu do gwiazdy (distance < threshold):
        * Płynne pojawienie się panelu HUD.
        * Wyświetla: Star_Class, Energy_Level, Brief, treść notatki.
        * Przycisk „Wejdź w szczegóły" – rozwija pełną treść w kokpicie.
        * Przycisk „Wróć" – zamknięcie panelu bez wychodzenia z 3D.
      - Styl: Cyberpunk, przezroczystość, animacje CSS.

   c) Drag&Drop Importer w Kokpicie:
      - Użytkownik przeciąga folder/plik na canvas 3D.
      - System pyta: „Zaimportować [nazwa] jako nowy sektor?"
      - Po akceptacji: LIBRARIAN skanuje, STORYTELLER analizuje, ASTRONOMER dodaje gwiazdy.

FAZA 5: AGRESYWNY PUSH (GIT-PUSHER + WATCHMAN)

   a) Push per Plik:
      - Po każdej udanej analizie pliku przez STORYTELLERA:
        * GIT-PUSHER: git add . → git commit -m "Galaxy Pilot: Mapping star [nazwa]" → git push
      - Kopia zapasowa postępów na bieżąco.

   b) Batch Push (fallback):
      - Jeśli push per plik zawodzi – batch co 10 plików.
      - WATCHMAN monitoruje i przełącza tryb.

   c) Pre-Push Security:
      - SECURITY-OFFICER waliduje .env/.gitignore.
      - DEBUGGER skanuje wycieki.
      - GIT-PUSHER wykonuje push tylko po zgodzie obu.

FAZA 6: NADZÓR I CIĄGŁOŚĆ (WATCHMAN + DEBUGGER)

   a) Monitoring API:
      - WATCHMAN liczy requesty/minutę.
      - Przy zbliżaniu się do limitu – zmniejsza batch size.
      - Automatyczne przełączanie local/API.

   b) Logi:
      - 00_Runtime/Galaxy_Build.log – log operacyjny.
      - 00_Runtime/INPUT_REQUIRED.md – problemy wymagające użytkownika.
      - data/output/progress.json – checkpointy dla wznowienia.

================================================================================
INSTRUKCJA WYKONAWCZA DLA HERMESA
================================================================================

- Działaj w pełnej autonomii jako Strateg.
- Deleguj zadania agentom przez Task tool.
- Monitoruj postęp przez WATCHMAN.
- Używaj wyłącznie cat << EOF do tworzenia plików (zakodowane w agentach).
- Nie przerywaj pracy do pełnego wdrożenia v2.0.
- Jeśli napotkasz problem – zapisz w 00_Runtime/INPUT_REQUIRED.md i kontynuuj.
- Wszystkie ścieżki Windows używają slashów: C:/Users/... (unikaj backslashy w YAML/JSON).

================================================================================
DRAG & DROP INTERFACE
================================================================================

W kokpicie 3D (dist/index.html) zaimplementuj:

1. Event listeners: dragover, dragleave, drop na canvasie Three.js.
2. Obsługę plików i folderów (webkitdirectory dla folderów).
3. Po dropie:
   - Odczytaj listę plików (rekurencyjnie dla folderów).
   - Filtruj: .md, .py, .js, .txt, .json (konfigurowalne).
   - Wyświetl modal: „Znaleziono X plików. Zaimportować?"
   - Po akceptacji: wyślij do backendu (fetch POST do lokalnego serwera) lub zapisz w IndexedDB.
4. Alternatywa bez backendu: zapisz w localStorage/IndexedDB i przeładuj galaxy_data.json.
